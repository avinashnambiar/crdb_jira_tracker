📄 How It Works

This is a zero-install standalone app. No browser extension needed.

  • All data is stored in your browser's localStorage — nothing leaves your machine.
  • The bookmarklet detects whether you're on a CRDB page or Jira page, extracts the relevant info, saves it via a tiny popup that auto-closes, and routes it to the correct tracker tab.
  • You can export/import your data as JSON from the Settings tab.

🔐 Jira API Token Setup (for auto bug status)

To automatically fetch and color-code Jira bug statuses, you need to create an Atlassian API Token and add it in Settings.

1
Go to Atlassian API Tokens and log in with your AMD Atlassian account
2
Click "Create API token", give it a label (e.g. CRDB Tracker), and copy the generated token
3
Open the Settings tab in this app and scroll to Jira Integration
4
Enter your Atlassian email (e.g. your.name@amd.com) and paste the API token
5
Click Save Settings — bug statuses will auto-fetch and show as colored badges
Security note: Your API token is stored only in your browser's localStorage — it never leaves your machine except when sent to Jira's API via the local proxy server.

⚡ Optional: Run with Local Server

For the most reliable experience (especially if localStorage acts up on file://), run a simple server:

  1. Open a terminal in this folder
  2. Run: python -m http.server 8080
  3. Open http://localhost:8080 in your browser
  4. Re-drag the bookmarklet from the Setup tab (it auto-updates the URL)
